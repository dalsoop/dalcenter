# Issue Lifecycle

이슈의 done criteria 생성과 닫기 전 검증을 담당하는 스킬.

## Done Criteria 생성 (이슈 할당 시)

이슈를 멤버에게 할당할 때, 이슈 본문을 분석하여 Done Criteria 체크리스트를 생성한다.

### 기본 항목

코드 변경이 포함된 이슈의 기본 체크리스트:

```markdown
## Done Criteria
- [ ] 코드 구현 + PR 생성
- [ ] CI 통과
- [ ] PR 머지
- [ ] 바이너리 빌드 + 배포 (코드 변경 시)
- [ ] 서비스 재시작 (필요 시)
- [ ] 실제 동작 확인
- [ ] 테스트 추가 (코드 변경 시)
- [ ] 문서 업데이트 (필요 시)
```

### 생성 규칙

1. 이슈 내용을 분석하여 필요한 done criteria 항목을 결정한다
2. 기본 템플릿에서 해당 이슈에 맞는 항목을 선택한다:
   - 코드 변경: 구현 + PR + CI + 머지 + 빌드/배포 + 테스트
   - 인프라 작업: 구축 + 설정 + 동작 확인
   - 문서 작업: 작성 + 리뷰 + 머지
3. 이슈 본문의 할 일 목록(`- [ ]`)을 기반으로 구체적 항목 추가
4. 이슈 성격에 따라 항목을 추가/제거한다
5. Done Criteria는 이슈 코멘트에 체크리스트로 작성한다

## Done Criteria 검증 (이슈 닫기 전)

이슈를 닫기 전 반드시 수행:

1. 이슈의 done criteria 체크리스트를 확인한다
2. 각 항목의 완료 여부를 실제로 검증한다:
   - PR 머지 여부: GitHub에서 확인
   - 빌드/배포: 바이너리 버전 또는 서비스 상태 확인
   - 동작 확인: 실제 기능 테스트
3. 검증된 항목만 체크한다
4. **모든 항목이 체크되어야만 이슈를 닫는다**

## 부분 완료 시 이슈 분리

미완료 항목이 있는데 일부 작업이 완료된 경우:

1. 완료된 항목을 체크한다
2. 남은 항목을 새 이슈로 생성한다:
   - 제목: `[Follow-up] 원본 이슈 제목 — 남은 작업`
   - 본문: 남은 done criteria 항목 나열
   - 원본 이슈 참조: `Follow-up from #원본번호`
3. 원본 이슈에 새 이슈 링크를 코멘트한다
4. 원본 이슈는 **닫지 않는다** — 새 이슈 완료 후 함께 닫는다

## GitHub Action (선택)

체크리스트 강제화를 자동화하려면 GitHub Action 도입을 검토할 수 있다:
- 이슈 close 이벤트 감지
- 본문의 `- [ ]` 미체크 항목이 있으면 close 차단 또는 리오픈
- 또는 dalcenter issue_watcher에서 close 이벤트 감지 후 미충족 시 리오픈
