# dalcenter `.dal/` 폴더 구조 및 `dal.spec.cue` 분석

## 1. 디렉토리 구조 개요

```
.dal/
├── dal.spec.cue              # CUE 스키마 — 모든 DAL 프로필의 타입 정의
├── context/                  # 대화 기록·컨텍스트 파일
├── data/                     # 런타임 데이터 (claims, escalations 등)
├── skills/                   # 공유 스킬 모듈 (12개)
│   ├── code-review/
│   ├── cue-schema/
│   ├── docker-ops/
│   ├── file-organize/
│   ├── go-ci/
│   ├── go-review/
│   ├── go-security/
│   ├── macos-launchagent/
│   ├── mattermost-api/
│   ├── security-audit/
│   ├── shell-script/
│   └── test-strategy/
├── leader/                   # DAL 프로필: leader
├── dev/                      # DAL 프로필: dev
├── reviewer/                 # DAL 프로필: reviewer
├── tester/                   # DAL 프로필: tester
├── verifier/                 # DAL 프로필: verifier
├── host/                     # DAL 프로필: host
├── codex-dev/                # DAL 프로필: codex-dev
└── mac-organizer/            # DAL 프로필: mac-organizer
```

각 DAL 프로필 디렉토리에는 `dal.cue` (프로필 정의)와 선택적으로 `instructions.md` (에이전트 지시문)가 포함된다.

## 2. `dal.spec.cue` — 핵심 타입 정의

```cue
#Player: "claude" | "codex" | "gemini"
#Role:   "leader" | "member"
#DalProfile: {
    uuid!:    string & != ""
    name!:    string & != ""
    version!: string
    player!:  #Player
    role!:    #Role
    skills?:  [...string]
    hooks?:   [...string]
}
```

### 2.1 `#Player` (열거형)

AI 엔진(플레이어) 타입을 제한한다.

| 값 | 설명 |
|---|---|
| `"claude"` | Anthropic Claude |
| `"codex"` | OpenAI Codex |
| `"gemini"` | Google Gemini |

### 2.2 `#Role` (열거형)

DAL의 조직 내 역할을 구분한다.

| 값 | 설명 |
|---|---|
| `"leader"` | 작업 배분·의사결정 권한을 가진 리더 |
| `"member"` | 지시를 받아 작업을 수행하는 멤버 |

### 2.3 `#DalProfile` (구조체)

각 에이전트의 신원과 역량을 선언하는 핵심 타입이다.

| 필드 | 필수 | 타입 | 설명 |
|---|---|---|---|
| `uuid` | **필수** | `string & != ""` | 고유 식별자 (예: `"dev-20260326"`) |
| `name` | **필수** | `string & != ""` | 사람이 읽을 수 있는 이름 |
| `version` | **필수** | `string` | 프로필 버전 (semver) |
| `player` | **필수** | `#Player` | 사용하는 AI 엔진 |
| `role` | **필수** | `#Role` | 조직 내 역할 |
| `skills` | 선택 | `[...string]` | 할당된 스킬 경로 목록 |
| `hooks` | 선택 | `[...string]` | 훅 식별자 목록 |

## 4. 스펙 외 확장 필드

`dal.spec.cue`의 `#DalProfile`에 정의되지 않았지만 실제 `dal.cue` 파일에서 사용되는 확장 필드들이 있다.

| 필드 | 사용 예 | 설명 |
|---|---|---|
| `git` | 전체 프로필 | `user`, `email`, `github_token` — Git 커밋·푸시에 사용 |
| `player_version` | verifier | `"go"` — 플레이어의 런타임 변형 지정 |
| `auto_task` | verifier | 자동 실행할 명령 문자열 |
| `auto_interval` | verifier | 자동 실행 주기 (예: `"1h"`) |

## 5. `data/` — 런타임 데이터 구조

### 5.1 `claims.example.json`

DAL이 호스트에게 보고하는 요청·이슈 레코드.

```json
{
  "id": "claim-NNNN",
  "dal": "<dal-name>",
  "type": "env | blocked | improvement | bug",
  "title": "...",
  "detail": "...",
  "context": "...",
  "timestamp": "ISO-8601",
  "status": "open | acknowledged | resolved",
  "response": "...",
  "responded_at": "ISO-8601"
}
```

### 5.2 `escalations.example.json`

DAL의 작업 실패·에스컬레이션 레코드.

```json
{
  "id": "esc-NNNN",
  "dal": "<dal-name>",
  "task": "...",
  "error_class": "instructions | env",
  "output": "...",
  "timestamp": "ISO-8601",
  "resolved": true | false,
  "resolved_at": "ISO-8601"
}
```
